## UI Elements Description

### Period Selection Controls
- **Dropdown 1**: "View variance for" with options:
  - Month (default)
  - Quarter
  - Year
- **Dropdown 2**: "Starting month" with scrollable list of available months
- Both dropdowns trigger dynamic updates via Turbo

### Variance Summary Card
- **Top Section**: 
  - Title: "Budget Variance Analysis"
  - Period label: "January 2024" (or "Q1 2024", "2024" based on selection)
- **Middle Section** (3-column grid):
  - Budgeted amount (large, bold)
  - Actual spent (large, bold)
  - Variance (colored based on status):
    - Green (+) for under budget
    - Red (-) for over budget
    - Shows both dollar amount and percentage
- **Progress Bar**:
  - Filled portion shows actual spending
  - Colors: Green (under budget) or Red (over budget)
  - Unfilled portion shows remaining budget
  - Percentage labels on both sides
  - Handles edge case: Shows "Over Budget" text when >100%

### Category Breakdown Table
- **Header**: "Category Breakdown"
- **Each Row**:
  - Color dot (category color)
  - Category name
  - Actual vs budgeted (e.g., "$450 / $500")
  - Variance amount with +/- sign
  - Color coding: Green for under, Red for over
- **Sorting**: By variance magnitude (largest variances first)
- **Limit**: Shows top 10 categories

## Color Scheme

- **Under Budget**: Green (#10B981)
- **Over Budget**: Red (destructive color from design system)
- **Neutral/Remaining**: Gray (surface-inset from design system)
- **Text Primary**: Standard text color
- **Text Secondary**: Subdued text color

## Responsive Behavior

- Desktop (md+): 3-column grid for summary stats
- Mobile: Single column stack
- Dropdowns: Full width on mobile
- Category list: Scrollable if many categories
